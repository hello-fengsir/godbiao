# 神奇阿标 (God Biao) — 产品介绍文档

> v1.0 | 2026年6月 | 作者：[峰Sir](https://github.com/hello-fengsir)

---

## 一、产品定位

**神奇阿标**是一款面向招投标场景的 AI 智能比对工具。它解决的问题非常聚焦：

> 投标文件几百页，招标要求几十条，人工逐条核对费时费力还容易出错。用 AI 自动读、自动比、自动出报告。

目标用户：
- 投标专员 / 标书制作人员
- 项目经理（最终审核标书）
- 小型投标团队（没有专门的标书审查岗）

---

## 二、核心功能

### 2.1 多格式标书解析

| 格式 | 支持情况 | 技术方案 |
|------|---------|---------|
| PDF | ✅ 文本 + 表格 + 内嵌图片 OCR | pdfplumber + pymupdf + tesseract |
| Word (.docx) | ✅ 段落 + 表格 + 内嵌图片 OCR | python-docx + zipfile |
| 图片 (.png/.jpg) | ✅ 纯图片 OCR | tesseract (chi_sim+eng) |
| 纯文本 (.txt) | ✅ | 直接读取 |

### 2.2 评标模式

| 模式 | 说明 |
|------|------|
| **合并评标** | 商务+技术在同一文件中，上传一个文件即可 |
| **分离评标** | 技术标和商务标分开上传，分别解析后合并比对 |

### 2.3 AI 三路并行评估

一次比对同时执行三个 AI 任务：

| 评估维度 | 输出 |
|---------|------|
| **内容比对** | 逐条判断「满足/部分满足/不满足」+ AI 总评 |
| **格式评估** | 格式评分卡（排版/字体/层级/签章）+ 问题清单 |
| **关键信息提取** | 投标方/招标方信息 + 商务关键项（金额/货期/付款/资质） |

### 2.4 多模型支持

内置 10 家主流大模型提供商预置配置，用户只需粘贴 API Key：

- DeepSeek（深度求索）
- OpenAI (GPT-4o / o4-mini)
- 通义千问（阿里云）
- 文心一言（百度）
- 混元（腾讯）
- 豆包（字节跳动）
- 智谱 GLM
- Kimi（月之暗面）
- MiniMax
- 商汤日日新

### 2.5 安全设计

- **API Key 零存储**：Key 仅保存在用户浏览器 localStorage，每次请求通过 HTTP Header 传递
- **服务器不落盘**：不上报、不记录、不持久化任何 API Key
- **自动清理**：刷新页面自动清除上传文件和过期记录

│  ┌─────────┐ ┌──────────┐ ┌──────────┐  │

        │           │             │
   ┌────▼───────────▼─────────────▼────┐
   │         FastAPI (Python)          │
   │  ┌──────────┐  ┌───────────────┐  │
   │  │ 文件解析  │  │  LLM 调用层   │  │

   └───────────────────────────────────┘

**技术栈**：
- 后端：Python 3.11 + FastAPI + Jinja2
- 解析：pdfplumber + pymupdf + python-docx + tesseract OCR
- 存储：SQLite（比对记录 1 天自动过期）
- 部署：Docker / Docker Compose

## 四、使用流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ ① 配 Key │ → │ ② 上传   │ → │ ③ 预览   │ → │ ④ 比对   │
│ 选模型    │    │ 标书+要求 │    │ 确认内容  │    │ AI 评估  │
└──────────┘    └──────────┘    └──────────┘    └────┬─────┘
                                                     │
                                              ┌──────▼─────┐
                                              │ ⑤ 查看结果  │
                                              │ 逐项+格式+  │
                                              │ 关键信息     │
                                              └────────────┘
```

1. **配置模型**：点击首页 ⚙️ 模型设置 → 选提供商 → 粘贴 Key → 测试连接 → 保存
2. **上传文件**：选择评标模式 → 上传标书 → 填写/上传招标要求
3. **预览确认**：检查 AI 提取的标书内容是否正确
4. **开始比对**：点击开始比对，实时进度条，三项并行评估
5. **查看结果**：
   - 左侧：逐项核查（满足/部分满足/不满足 + 百分比统计）
   - 右侧：格式评分卡 + 关键信息对比卡
   - 底部：AI 总评

## 五、部署方式

### Docker 部署（推荐）

```bash
git clone https://github.com/hello-fengsir/godbiao.git
cd godbiao
cp .env.example .env
# 可选：编辑 .env 填入默认 API Key
docker compose up -d
```

访问 `http://localhost:8880`

### 本地开发

```bash
pip install fastapi uvicorn jinja2 python-multipart \
    pdfplumber python-docx pytesseract pillow pymupdf \
    aiofiles aiosqlite httpx
python app.py
```

需要系统安装 tesseract-ocr 及中文语言包。

## 六、设计原则

1. **先通后优**：功能优先跑通，再迭代优化
2. **安全第一**：API Key 绝不落服务器，用户完全掌控
3. **离线友好**：所有处理在本地完成，不上传文件到第三方
4. **模型无关**：支持任意 OpenAI 兼容 API，不锁定供应商
5. **简单即美**：三页面完成全流程，无冗余操作
